Migración: Competencias

Tabla: competencias
Representa el evento principal que agrupa uno o varios torneos independientes. Contiene únicamente la información general de la competencia.
    id
    tipo_competencia_id -> foreignId()->constrained('competencia_tipos')
    creador_id -> foreignId()->constrained('users')
    nombre -> string
    descripcion -> text()->nullable()
    es_nocturna -> boolean()->default(false)
    estatus -> tinyInteger()->default(0)
        0 Configuración
        1 Activa
        2 Suspendida
        3 Finalizada
        4 Cancelada
    timestamps()
Nota: El estatus representa el estado general de la competencia como evento. Puede calcularse automáticamente con base en el estado de sus grupos o administrarse manualmente por el organizador.

Tabla: competencia_grupos
Representa cada torneo independiente generado dentro de una competencia a partir de la combinación de Género + Categoría + División.
Cada registro posee su propia configuración y puede desarrollarse de forma independiente del resto de los grupos pertenecientes a la misma competencia.
    id
    competencia_id -> foreignId()->constrained('competencias')
    genero_id -> foreignId()->constrained('competencia_generos')
    categoria_id -> foreignId()->constrained('competencia_categorias')
    division_id -> foreignId()->constrained('competencia_divisiones')
    fecha_inicio -> date
    fecha_fin -> date
    estatus -> tinyInteger()->default(0)
        0 Configuración
        1 Activa
        2 Suspendida
        3 Finalizada
        4 Cancelada
        timestamps()

Generación automática de grupos
Supongamos que el administrador registra la siguiente competencia:

Liga Municipal 2027

Géneros
✓ Varonil
✓ Femenil

Categorías
✓ Libre
✓ Juvenil

Divisiones
✓ Primera
✓ Segunda

El sistema generará automáticamente ocho registros en competencia_grupos.

Liga Municipal
└── Varonil
    ├── Libre
    │   ├── Primera
    │   └── Segunda
    │
    └── Juvenil
        ├── Primera
        └── Segunda

Liga Municipal
└── Femenil
    ├── Libre
    │   ├── Primera
    │   └── Segunda
    │
    └── Juvenil
        ├── Primera
        └── Segunda

Cada registro funcionará como un torneo completamente independiente, compartiendo únicamente la información general de la competencia.

Toda la operación del sistema dependerá de CompetenciaGrupo.

Competencias
      │
      ▼
CompetenciaGrupos
      │
      ├── Equipos
      ├── Jugadores
      ├── Cuerpos Técnicos

Filosofía del proyecto

La entidad Competencia representa únicamente el contenedor principal del torneo.

La entidad CompetenciaGrupo representa la unidad de negocio real sobre la que operan todos los módulos del sistema.

Esta arquitectura permite:

Escalar una competencia a múltiples categorías sin duplicar información.
Asumir configuraciones unicas entre los grupos.
Administrar cada grupo como un torneo independiente.
Mantener una estructura modular, limpia y preparada para futuras funcionalidades como estadísticas, inscripciones, pagos, arbitraje y reportes.
